De-Encapsulation Example

This process is reversed at the receiving host, and is known as de-encapsulation. De-encapsulation is the process used by a receiving device to remove one or more of the protocol headers. The data is de-encapsulated as it moves up through the layers towards the end-user application.

Click Play in the figure to see the de-encapsulation process.

The figure on this page is an animation that shows the following de-encapsulation process as the web client receives the web page from the web server. Step 1: When the Ethernet Frame is received by the web client the Frame Header and Frame Trailer are read by the Data Link Layer. Step 2: The Data Link layer sends the IP Packet to the Network Layer. Step 3: The Network Header is read by the Network Layer. Step 4: The network layer sends the TCP Segment to the Transport Layer. Step 5: The Transport Header is read by the Transport Layer. Step 6: The data is available to the end-user application.
